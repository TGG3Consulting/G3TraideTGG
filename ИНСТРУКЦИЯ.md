## 1. Что это такое

**BinanceFriend** — это система обнаружения манипуляций (pump-and-dump) на Binance.

### Задача системы:
- Найти монеты, которые ЛЕГКО манипулировать (низкая ликвидность)
- Мониторить их в реальном времени
- Детектировать подозрительную активность
- Отправлять алерты в Binance (когда дадут API)

### Что система НЕ делает:
- НЕ торгует
- НЕ манипулирует рынком
- НЕ требует API ключей для торговли (только для отправки алертов Binance)

## 3. Входные данные

### Система получает данные АВТОМАТИЧЕСКИ из публичного API Binance:

| Данные | Источник | Описание |
|--------|----------|----------|
| Список пар | REST API | Все торговые пары (~1500) |
| 24h статистика | REST API | Объём, цена, количество сделок |
| Глубина стакана | REST API | Bid/Ask на 100 уровней |
| Сделки | WebSocket | Каждая сделка в реальном времени |
| Стакан | WebSocket | Изменения каждые 100мс |
| Свечи | WebSocket | Минутные свечи |

### Вам НЕ нужно:
- Вводить список монет (система сама найдёт уязвимые)
- Настраивать пороги (уже настроены)
- Запускать что-то вручную (работает автоматически)

## 5. Настройка Telegram

### 5.1 Создание бота

1. Откройте Telegram и найдите **@BotFather**
2. Отправьте команду `/newbot`
3. Введите имя бота (например: `BinanceFriend Alerts`)
4. Введите username бота (например: `binancefriend_alerts_bot`)
5. **BotFather** выдаст вам **token** вида:
   ```
   7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```

### 5.2 Получение Chat ID

**Способ 1: Личные сообщения**
1. Напишите вашему боту любое сообщение
2. Откройте в браузере:
   ```
   https://api.telegram.org/bot<ВАШ_TOKEN>/getUpdates
   ```
3. Найдите в ответе `"chat":{"id":123456789}` — это ваш chat_id

**Способ 2: Группа/канал**
1. Добавьте бота в группу/канал как администратора
2. Напишите любое сообщение в группу
3. Откройте тот же URL (getUpdates)
4. Chat ID группы будет отрицательным: `-1001234567890`

### 5.3 Настройка конфига

Откройте файл `config/telegram.json` и введите данные:

```json
{
    "bot_token": "7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
    "chat_id": "123456789"
}
```

**Или через командную строку:**

```bash
py -3.11 run.py --telegram-token "7123456789:AAH..." --telegram-chat "123456789"
```

## 6. Конфигурация и ключи

### 6.1 Где хранится конфигурация

```
G:\BinanceFriend\config\config.yaml
```

### 6.2 Ключи Binance

**ВАЖНО:** Для РАБОТЫ скринера ключи НЕ нужны!
Ключи нужны ТОЛЬКО для отправки алертов в Binance API (когда они его дадут).

Когда Binance даст API endpoint, добавьте в `config/config.yaml`:

```yaml
alerts:
  # URL который даст Binance
  binance_api_url: "https://api.binance.com/sapi/v1/manipulation-alerts"

  # Ключи которые даст Binance
  api_key: "ваш_api_key_от_binance"
  api_secret: "ваш_api_secret_от_binance"
```

### 6.3 Основные настройки

```yaml
# config/config.yaml

screener:
  # Как часто пересканировать все пары (секунды)
  rescan_interval: 300  # 5 минут

  # Сколько пар мониторить одновременно
  max_symbols: 100

filter:
  # Максимальный объём для "уязвимой" пары
  max_volume_usd: 1000000  # $1M

  # Максимальная глубина стакана
  max_depth_usd: 50000  # $50K

detection:
  # Пороги для детекции
  volume_spike_warning: 10   # 10x = WARNING
  volume_spike_alert: 20     # 20x = ALERT
  volume_spike_critical: 50  # 50x = CRITICAL
```

### 6.4 Альтернатива: параметры командной строки

Вместо редактирования config.yaml можно передать параметры при запуске:

```bash
py -3.11 run.py --max-symbols 50 --rescan-interval 120
```

## 7. Запуск

### 7.1 Простой запуск

```bash
cd G:\BinanceFriend
py -3.11 run.py
```

### 7.2 Запуск с параметрами

```bash
# Мониторить только 50 пар, сканировать каждые 2 минуты
py -3.11 run.py --max-symbols 50 --rescan-interval 120

# С указанием API Binance (когда дадут)
py -3.11 run.py --api-url "https://..." --api-key "..." --api-secret "..."
```

### 7.3 Все параметры

```
--max-symbols N       Максимум пар для мониторинга (default: 100)
--rescan-interval N   Интервал сканирования в секундах (default: 300)
--api-url URL         URL Binance API для алертов
--api-key KEY         API ключ
--api-secret SECRET   API секрет
--log-file PATH       Путь к файлу логов (default: logs/alerts.jsonl)
--log-level LEVEL     Уровень логов: DEBUG, INFO, WARNING, ERROR
--json-logs           Логи в JSON формате
--telegram-token TOK  Telegram bot token
--telegram-chat ID    Telegram chat ID
```

### 7.4 Остановка

Нажмите `Ctrl+C` — система корректно завершится.

## 8. Примеры алертов

### Пример 1: Volume Spike

```
⚠️  DETECTION: LOWCAPUSDT
   type=VOLUME_SPIKE_HIGH
   score=75
   severity=ALERT
   • Volume 25.3x above average
   • 5-min volume: $156,000
   • Average 5-min: $6,200
```

**Что это значит:** За последние 5 минут объём торгов в 25 раз выше обычного. Кто-то активно торгует этой монетой.

### Пример 2: Active Pump

```
🔴 DETECTION: SCAMUSDT
   type=ACTIVE_PUMP
   score=98
   severity=CRITICAL
   • ACTIVE PUMP DETECTED
   • Volume: 67.4x normal
   • Price: +45.2% in 5min
   • Book: 91% imbalanced
   • Trade flow: 94% buys
```

**Что это значит:** Классический pump:
- Объём взлетел в 67 раз
- Цена выросла на 45% за 5 минут
- Стакан перекошен в сторону покупок
- 94% сделок — агрессивные покупки

### Пример 3: Wash Trading

```
🚨 DETECTION: FAKEUSDT
   type=WASH_TRADING_LIKELY
   score=90
   severity=CRITICAL
   • 62% of trades have identical size
   • Quantity 1000.0 repeated 45 times
   • Total 72 trades in 5 minutes
```

**Что это значит:** Кто-то торгует сам с собой (wash trading):
- 62% сделок одинакового размера (1000 монет)
- Это искусственно создаёт объём

## 9. FAQ

### Q: Нужен ли VPN?
**A:** Нет, если Binance не заблокирован в вашей стране.

### Q: Сколько ресурсов потребляет?
**A:** Минимально:
- CPU: 1-5%
- RAM: ~100-200 MB
- Сеть: ~1-5 Мбит/с (WebSocket)

### Q: Можно запустить 24/7?
**A:** Да, система сама переподключается при обрывах.

### Q: Что если Binance заблокирует IP?
**A:** Система использует только публичный API без ключей, лимиты очень высокие. При проблемах — использовать VPN.

### Q: Где смотреть результаты?
**A:**
1. В консоли (в реальном времени)
2. В файле `logs/alerts.jsonl`

### Q: Как отправить алерты в Binance?
**A:** Когда Binance даст API endpoint, добавить в config:
```yaml
alerts:
  binance_api_url: "URL_от_Binance"
  api_key: "ключ_от_Binance"
  api_secret: "секрет_от_Binance"
```

### Q: Что делать если много ложных срабатываний?
**A:** Увеличить пороги в `config/config.yaml`:
```yaml
detection:
  volume_spike_warning: 15   # было 10
  volume_spike_alert: 30     # было 20
```
