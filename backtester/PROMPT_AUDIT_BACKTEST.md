## СПИСОК ПАТТЕРНОВ (начальный: 15 штук, может расти)

При анализе КАЖДОГО файла проверяются ВСЕ существующие паттерны.
Список паттернов **НЕ ФИКСИРОВАН** — он может расти при обнаружении новых закономерностей.

### P01: Понедельник убыточен
**Условие TRUE:** Пн PnL < 0 И Пн даёт > 50% общих потерь
**Что означает:** В понедельник рынок нестабилен после выходных. Накопившиеся ордера создают хаос, ложные пробои.
**Действие если TRUE:** Не торговать в понедельник или ужесточить фильтры.

### P02: Prob 55 лучше Prob 50
**Условие TRUE:** PnL(Prob55) > PnL(Prob50)
**Что означает:** Сигналы с вероятностью 55+ качественнее чем минимальные 50.
**Действие если TRUE:** Поднять минимальный порог probability до 55.

### P03: Score 50+ лучше 45-50
**Условие TRUE:** PnL(Score≥50) > PnL(Score 45-50)
**Что означает:** Accumulation Score — уверенность генератора. Низкий score = слабые сигналы.
**Действие если TRUE:** Поднять min_accumulation_score до 50-55.

### P04: OI_SPIKE прибылен
**Условие TRUE:** PnL(OI_SPIKE триггер) > 0
**Что означает:** Резкий рост открытого интереса = новые деньги входят в рынок. Сильный сигнал.
**Действие если TRUE:** Приоритизировать сигналы с OI_SPIKE.

### P05: COORDINATED_BUYING убыточен
**Условие TRUE:** PnL(COORDINATED_BUYING триггер) < 0
**Что означает:** "Координированная покупка" часто = ложный сигнал или запоздалый вход.
**Действие если TRUE:** Пересмотреть логику триггера или снизить его вес.

### P06: NO_TRIGGER лучше триггеров
**Условие TRUE:** PnL(NO_TRIGGER) > PnL(лучший триггер)
**Что означает:** Сигналы БЕЗ триггеров работают лучше. Триггеры добавляют шум.
**Действие если TRUE:** Пересмотреть систему триггеров.

### P07: Volume Spike 0-0.5 прибылен
**Условие TRUE:** PnL(Volume Spike 0-0.5) > 0
**Что означает:** Низкий объём = "тихое накопление". Умные деньги входят незаметно.
**Действие если TRUE:** Искать сигналы с низким volume spike.

### P08: Volume Spike 2.0+ убыточен
**Условие TRUE:** PnL(Volume Spike 2.0+) < 0
**Что означает:** Очень высокий объём = FOMO/паника. Входить = покупать на хаях.
**Действие если TRUE:** Фильтровать сигналы с spike > 2.0.

### P09: Funding отрицательный лучше
**Условие TRUE:** PnL(Funding < 0) > PnL(Funding ≥ 0)
**Что означает:** Отрицательный фандинг = толпа в шортах, лонги дешёвые. Контрарный сигнал.
**Действие если TRUE:** Входить в лонг при отрицательном фандинге.

### P10: 1 монета даёт >80% убытков
**Условие TRUE:** Худшая монета > 80% Total Loss
**Что означает:** Одна "токсичная" монета убивает всю выборку.
**Действие если TRUE:** Blacklist монет или индивидуальные настройки.

### P11: Часы 10-12 UTC убыточны
**Условие TRUE:** PnL(10:00 + 11:00 + 12:00) < 0
**Что означает:** Открытие Европы. Много шума, ложных движений.
**Действие если TRUE:** Не торговать 10-12 UTC.

### P12: Часы 19-21 UTC прибыльны
**Условие TRUE:** PnL(19:00 + 20:00 + 21:00) > 0
**Что означает:** Разгар американской сессии. Тренды сформированы.
**Действие если TRUE:** Приоритизировать сигналы в 19-21 UTC.

### P13: TP3 rate > 8%
**Условие TRUE:** TP3_hits / filled > 8%
**Что означает:** Как часто сделки доходят до полного тейка.
**Действие если FALSE:** TP слишком далеко или стратегия слабая.

### P14: SL rate > 60%
**Условие TRUE:** SL_exits / filled > 60%
**Что означает:** Более 60% сделок закрываются по стопу. Это плохо.
**Действие если TRUE:** Пересмотреть SL% или качество сигналов.

### P15: Avg hold WINS > LOSSES
**Условие TRUE:** mean(hours_wins) > mean(hours_losses)
**Что означает:** Прибыльные сделки держатся дольше. Это норма.
**Действие если TRUE:** Всё ок, не требует изменений.

## ПОРЯДОК РАБОТЫ

### При анализе НОВОГО файла:

1. Запустить `python audit_backtest.py "путь_к_файлу.xlsx"`
2. Получить сырые данные из вывода скрипта
3. Проверить ВСЕ 15 паттернов по условиям выше
4. Прочитать текущую таблицу из `PATTERNS_TABLE.md`
5. Добавить новую строку в таблицу
6. Пересчитать статистику (% TRUE для каждого паттерна)
7. Сохранить обновлённую `PATTERNS_TABLE.md`
8. Вывести результат пользователю

### При обнаружении НОВОГО паттерна (P16, P17...):

**КОГДА ДОБАВЛЯТЬ:** Если при анализе файла обнаружена явная закономерность, которая:
- Повторяется в данных (не единичный случай)
- Имеет логическое объяснение (почему так происходит)
- Может привести к конкретному действию (фильтр, настройка)

**ПОРЯДОК ДЕЙСТВИЙ:**

1. **НЕ ТРОГАТЬ существующие паттерны P01-P15** — они остаются как есть
2. Создать новый паттерн с номером P16 (или следующий свободный)
3. Описать в формате:
   ```
   ### P16: [Название]
   **Условие TRUE:** [точное условие]
   **Что означает:** [объяснение на человеческом языке]
   **Действие если TRUE:** [конкретное действие]
   ```
4. Добавить описание в этот промпт (в раздел паттернов)
5. **ОБЯЗАТЕЛЬНО: Переанализировать ВСЕ ранее проанализированные файлы** на предмет нового паттерна
6. Добавить новую колонку P16 в таблицу `PATTERNS_TABLE.md`
7. Заполнить значения P16 для ВСЕХ файлов (включая старые)
8. Пересчитать статистику
9. Только после этого делать выводы о паттерне

**ВАЖНО:** Новый паттерн считается валидным только после проверки на ВСЕХ файлах!
