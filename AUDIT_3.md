## КРИТИЧЕСКАЯ ПРОБЛЕМА: РАЗНЫЕ СИГНАЛЫ В ЛОГАХ И TELEGRAM

**Суть жалобы пользователя:** "В логах одни сигналы, в Telegram другие"

**ПРИЧИНА НАЙДЕНА:** Это НЕ баг — это **ДВА РАЗНЫХ ТИПА СООБЩЕНИЙ**:

| Тип | Метод отправки | Чат Telegram | Что отправляется |
|-----|----------------|--------------|------------------|
| **ДЕТЕКЦИИ** | `send_alert()` | `-1003883956458` (основной бот) | "🚨 АЛЕРТЫ МАНИПУЛЯЦИЙ" — БЕЗ entry/SL/TP |
| **ТОРГОВЫЕ СИГНАЛЫ** | `send_trade_signal()` | `-5120007306` (signals бот) | Полные сигналы с entry/SL/TP |

**Пользователь скорее всего смотрит на ОДИН чат и видит:**
- Много детекций (ACTIVE_PUMP, VOLUME_SPIKE, etc.) — это НЕ торговые сигналы
- Мало торговых сигналов (LONG/SHORT с уровнями) — это НАСТОЯЩИЕ сигналы

**Файл `signals.jsonl` содержит ТОЛЬКО торговые сигналы, а не детекции!**

## 2. НАЙДЕННЫЕ ПРОБЛЕМЫ

### 2.1 БАГ: Cross-exchange детекции НЕ генерируют торговые сигналы

**Файл:** `src/screener/screener.py`
**Строки:** 727-821

**Проблема:**
Метод `_handle_cross_detection()` НЕ вызывает `signal_generator.on_detection()`.

**Сравнение:**

```python
# _handle_detection() — ПРАВИЛЬНО (строки 537-589):
if self.signal_generator and detection.severity.value >= AlertSeverity.ALERT.value:
    signal = self.signal_generator.on_detection(detection)
    if signal:
        self.signal_logger.log_signal(signal, ...)
        self.telegram_notifier.send_trade_signal(signal_text)

# _handle_cross_detection() — ПРОБЛЕМА (строки 814-821):
await self.alert_dispatcher.dispatch(screener_detection)
if self.telegram_notifier:
    asyncio.create_task(self.telegram_notifier.send_alert(screener_detection))
# ❌ НЕТ ВЫЗОВА signal_generator.on_detection()!
# ❌ НЕТ ЛОГИРОВАНИЯ В signals.jsonl!
# ❌ НЕТ send_trade_signal()!
```

**Последствие:**
Детекции типа `CX-001_PRICE_DIVERGENCE`, `CX-003_FUNDING_ARBITRAGE`, `CX-004_OI_MIGRATION`:
- Отправляются в Telegram как детекции (send_alert)
- НЕ генерируют торговые сигналы
- НЕ логируются в signals.jsonl
- Хотя они указаны в `_signal_triggers` в signal_generator.py!

---

### 2.2 Несоответствие: _signal_triggers содержит cross-exchange типы, но они не используются

**Файл:** `src/signals/signal_generator.py`
**Строки:** 128-131

```python
# Cross-exchange signals — УКАЗАНЫ, но НИКОГДА НЕ ВЫЗЫВАЮТСЯ!
"CX-001_PRICE_DIVERGENCE": SignalType.CROSS_EXCHANGE,
"CX-003_FUNDING_ARBITRAGE": SignalType.CROSS_EXCHANGE,
"CX-004_OI_MIGRATION": SignalType.CROSS_EXCHANGE,
```

Эти типы НЕ будут работать потому что:
1. Cross-exchange детекции имеют тип `CROSS_CX-001_PRICE_DIVERGENCE` (с префиксом `CROSS_`)
2. Но `_handle_cross_detection()` вообще не вызывает `on_detection()`

**Результат:** Статистика детекций завышена для futures.

## 9. РЕЗЮМЕ

### Почему "в логах одни сигналы, в Telegram другие":

1. **В Telegram ДВА чата:**
   - Основной чат получает ДЕТЕКЦИИ (много)
   - Сигнальный чат получает ТОРГОВЫЕ СИГНАЛЫ (мало)

2. **signals.jsonl содержит ТОЛЬКО торговые сигналы, не детекции**

3. **Cross-exchange детекции вообще не генерируют торговые сигналы** — это БАГ

### Что нужно сделать пользователю:

1. Проверить ОРИГИНАЛ сигнального чата (-5120007306) — там должны быть ТЕ ЖЕ сигналы что в signals.jsonl
2. Не путать детекции (🚨 АЛЕРТЫ МАНИПУЛЯЦИЙ) с торговыми сигналами (💰 ТОРГОВЫЙ СИГНАЛ)

### Что нужно исправить в коде:

1. **КРИТИЧНО:** Добавить генерацию торговых сигналов в `_handle_cross_detection()`
2. Убрать дублирование статистики
3. Исправить кодировку UTF-8
