## 3. КЛЮЧЕВОЕ РАЗЛИЧИЕ: ДЕТЕКЦИИ vs СИГНАЛЫ

### 3.1 ДЕТЕКЦИИ (send_alert)

```
🚨 АЛЕРТЫ МАНИПУЛЯЦИЙ (3 шт.)
⏰ 14:12:18

1. 🔴 PEPEUSDT
   Экстремальный всплеск объёма
   [█████] 95/100
   └ Объём: 454.7x

2. ⚠️ ATMUSDT
   ...
```

- Отправляются для КАЖДОЙ детекции (много)
- Не содержат entry/SL/TP
- Идут в основной чат

### 3.2 ТОРГОВЫЕ СИГНАЛЫ (send_trade_signal)

```
💰 ТОРГОВЫЙ СИГНАЛ

📈 LONG PEPEUSDT
🎯 Вероятность: 65%

📍 Вход: 0.00000430 - 0.00000434
⛔ Стоп: 0.00000400 (-7.0%)
✅ TP1: 0.00000500 (+15.0%) [30%]
✅ TP2: 0.00000700 (+62.0%) [40%]
✅ TP3: 0.00000900 (+108%) [30%]

R:R = 3.0
```

- Генерируются ТОЛЬКО если signal != None (мало)
- Содержат полную торговую информацию
- Идут в отдельный сигнальный чат

## 4. АНАЛИЗ: ПОЧЕМУ ПОЛЬЗОВАТЕЛЬ ВИДИТ "РАЗНЫЕ" СИГНАЛЫ

### Гипотеза 1: Пользователь смотрит НЕ тот чат

| Чат | ID | Что содержит |
|-----|-----|-------------|
| Основной | `-1003883956458` | ДЕТЕКЦИИ (много) |
| Сигналы | `-5120007306` | ТОРГОВЫЕ СИГНАЛЫ (мало) |

**Вывод:** Нужно смотреть чат `-5120007306` для сравнения с `signals.jsonl`

### Гипотеза 2: Пользователь путает детекции и сигналы

- Детекции приходят чаще (каждая манипуляция)
- Сигналы приходят реже (только если score >= 60, severity >= ALERT, R:R >= 2.0)

### Гипотеза 3: Telegram отправка может фейлиться

**Проверка:** В `send_trade_signal()` есть логирование:
- `"sending_trade_signal"` — перед отправкой
- `"trade_signal_sent_to_telegram"` — при успехе
- `"trade_signal_send_failed"` — при ошибке

## 8. РЕЗЮМЕ

### Проблема пользователя "в логах одни, в Telegram другие":

**НЕ БАГ, А НЕПОНИМАНИЕ АРХИТЕКТУРЫ:**

1. **В Telegram ДВА ЧАТА:**
   - Основной чат — получает ДЕТЕКЦИИ (формат "🚨 АЛЕРТЫ")
   - Сигнальный чат — получает ТОРГОВЫЕ СИГНАЛЫ (формат "💰 LONG/SHORT")

2. **`signals.jsonl` = только торговые сигналы**

3. **Детекции >> Сигналы по количеству**
   - Детекции: каждая манипуляция
   - Сигналы: только если проходят все фильтры

### Что нужно проверить пользователю:

1. **Открыть ПРАВИЛЬНЫЙ чат** — чат с ID `-5120007306`
2. **Сравнить timestamp'ы** — сигналы в jsonl и в Telegram должны совпадать
3. **Посмотреть логи** — `grep "trade_signal_sent_to_telegram" logs/console.log`

### Если сигналы всё ещё не совпадают:

```bash
# Посмотреть отправленные сигналы
grep -E "trade_signal_sent|sending_trade_signal|trade_signal_send_failed" logs/console.log

# Посмотреть залогированные сигналы
wc -l logs/signals.jsonl
```
